# Technical-Interview-Trainer

A mock technical interview simulator that watches you code and listens to you think. It runs an AI interviewer in real time, asks clarifying questions, offers hints when you're stuck, and gives you a structured coaching debrief at the end.

Built because generic LeetCode grinding doesn't teach you how to actually perform in an interview with the communication, the thinking out loud. This tries to replicate that.

---
## How it works

There are three stages:

**Setup.** Paste in any problem: a LeetCode problem, something from a company prep guide, whatever you're working on. 

**Live interview.** A split-screen view with a code editor on the left and a conversation panel on the right. You talk through your approach as you code, the same way you would in a real interview. The AI listens via your microphone, responds as an interviewer would (asking about complexity, probing edge cases, offering a nudge if you've been silent and stuck for a while), and speaks its responses back to you via text-to-speech. When you're done, you either click "End Interview" or the AI will wrap up naturally.

**Debrief.** After the session ends, the full transcript and your final code get sent to a separate analysis call that produces structured feedback: correctness, complexity analysis, communication quality, what went well, and concrete things to work on. The goal is feedback you can actually act on, not a score.

## Tech stack

**Frontend** — React with Vite, Monaco Editor (the VS Code engine) for the code panel, Tailwind CSS for styling.

**Backend** — FastAPI.

**Speech-to-text** — [faster-whisper](https://github.com/SYSTRAN/faster-whisper) running locally, using the `large-v3` model by default. The problem title gets injected as an initial prompt so Whisper handles things like "binary search tree" or "dynamic programming" without mangling them. Runs on CPU by default.

**Text-to-speech** — [Kokoro](https://github.com/hexgrad/kokoro), an open-source Apache 2.0 model. Runs on CPU by default.

**LLM** — Claude (claude-sonnet-4-6) via the Anthropic API, used for both the live interviewer and the post-session debrief.

## Setup

You'll need 3.10 <= Python < 3.13 and Node 18+ installed before starting.

**Backend**

```bash
cd backend
python -m venv venv
source venv/bin/activate  # Windows: venv\Scripts\activate
pip install -r requirements.txt
```

Create a `.env` file inside `backend/` with your Anthropic API key:

```
ANTHROPIC_API_KEY=your_key_here
```

Then start the server:

```bash
uvicorn main:app --reload
```

The backend runs on `http://localhost:8000` by default.

**Frontend**

```bash
cd frontend
npm install
npm run dev
```

The frontend runs on `http://localhost:5173` by default.

## API endpoints

`POST /transcribe` — Accepts a `.webm` audio blob, returns a transcript string. Whisper runs server-side with the problem terms injected as context.

`POST /chat` — Accepts the conversation history, current code state, and problem description. Returns the interviewer's next message as text.

`POST /speak` — Accepts a text string, returns an MP3 audio blob generated by Kokoro.

`POST /debrief` — Accepts the full transcript and final code. Returns structured markdown feedback across five coaching dimensions.

## GPU acceleration

Both faster-whisper and Kokoro run on CPU by default, which works fine but adds noticeable latency — transcription in particular can take longer than the audio itself on slower machines. Both can be switched to GPU, which typically brings transcription down to near real-time and TTS down to under half a second.

The setup process for GPU support varies significantly depending on your hardware.

**NVIDIA** — Install the CUDA toolkit, then change the `device` argument in the faster-whisper and Kokoro initialisations from `cpu` to `cuda`. For faster-whisper, also set `compute_type="float16"` for best performance.

**AMD** — Requires installing ROCm-enabled PyTorch wheels from AMD's package repository and a ROCm-compiled build of CTranslate2 (which faster-whisper depends on) from the OpenNMT GitHub releases. AMD's official documentation at `rocm.docs.amd.com` is the most reliable reference. The code changes after setup are the same as NVIDIA — `device="cuda"` in both places, since ROCm maps itself to the CUDA interface.

## Whisper model size

The default `large-v3` model gives the best transcription accuracy, especially for technical vocabulary, but it requires around 3GB of VRAM (or system RAM on CPU) and is slower on modest hardware. If you're running on CPU or a GPU with limited memory, faster-whisper supports several smaller variants that trade some accuracy for significantly lower resource usage.

To change the model, find the WhisperModel initialisation in your backend and swap out the model name:

```python
# Default — best accuracy, highest resource usage (~3GB)
model = WhisperModel("large-v3", device="cpu")

# Good middle ground — noticeably faster, still solid accuracy (~1.5GB)
model = WhisperModel("medium", device="cpu")

# Lightweight — fast even on weak hardware, accuracy drops on heavy accents
# and uncommon technical terms (~400MB)
model = WhisperModel("base", device="cpu")
```

The `initial_prompt` injection (which feeds the problem's technical terms into Whisper before transcribing) helps compensate for accuracy loss on smaller models, so the drop-off in practice is less severe than the benchmarks suggest.
